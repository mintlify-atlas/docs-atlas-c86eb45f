## What is MkDowner?

MkDowner is a modern React web application that converts various document formats into clean, readable Markdown. Built with a React frontend and powered by Microsoft MarkItDown on the backend, it provides a seamless drag-and-drop experience for document conversion.

## Key Features

### Drag & Drop Interface

MkDowner features an intuitive drag-and-drop upload area built with React. Simply drop your files or click to browse:

```tsx
const handleDrop = (e: React.DragEvent) => {
  e.preventDefault();
  setIsDragOver(false);
  if (e.dataTransfer.files) {
    onFilesSelected(e.dataTransfer.files);
  }
};
```

### Real-Time Progress Tracking

Watch your files convert with live progress indicators:

```tsx
const progressInterval = setInterval(() => {
  setProgress(prev => {
    const newProgress = prev + Math.random() * 15;
    return newProgress >= 90 ? 90 : newProgress;
  });
}, 200);
```

### Automatic Downloads

Converted files download automatically - single files as `.md`, multiple files as `.zip`:

```tsx
const fileName = files.length === 1 ? 
  `${files[0].name.split('.')[0]}.md` : 
  'converted_files.zip';
```

## Architecture Overview

MkDowner follows a modern, modular architecture:

### Frontend (React + TypeScript + Vite)

```
src/
├── components/          # Modularized React components
│   ├── UploadArea/     # Drag & drop upload interface
│   ├── FileList/       # Selected files display
│   ├── Header/         # App header and branding
│   └── SupportedFormats/ # Format information
├── hooks/              # Custom React hooks
│   └── useFileUpload.ts # File handling logic
├── services/           # API communication
│   └── api.ts         # Backend integration
└── App.tsx            # Main application component
```

### Backend (Python + Microsoft MarkItDown)

The backend API handles file conversion through a simple REST endpoint:

```typescript
const response = await fetch(`${API_BASE_URL}/upload`, {
  method: 'POST',
  body: formData,
});
```

<Info>
  The frontend communicates with the backend via the `/upload` endpoint, which processes files and returns converted Markdown or ZIP archives.
</Info>
